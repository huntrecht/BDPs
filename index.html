BDPs

Brokers Design Proposals (BDPs) describe standards for the Brokers digital platform, including core protocol specifications, client APIs, and contract standards.

Contributing

First review BDP-1. Then clone the repository and add your BDP to it. There is a template BDP here. Then submit a Pull Request to Huntrecht's BDPs repository.

BDP status terms

  • Draft - an BDP that is open for consideration and is undergoing rapid iteration and changes.
  • Last Call - an BDP that is done with its initial iteration and ready for review by a wide audience.
  • Accepted - a core BDP that has been in Last Call for at least 2 weeks and any technical changes that were requested have been addressed by the author. The process for Core Devs to decide whether to encode an BDP into their clients as part of a hard fork is not part of the BDP process. If such a decision is made, the BDP will move to final.
  • Final (non-Core) - an BDP that has been in Last Call for at least 2 weeks and any technical changes that were requested have been addressed by the author.
  • Final (Core) - an BDP that the Core Devs have decided to implement and release in a future hard fork or has already been released in a hard fork.

BDP Types

BDPs are separated into a number of types, and each has its own list of BDPs.

Standard Track ({{site.pages|where:"type","Standards Track"|size}})

Describes any change that affects most or all Brokers implementations or any change or addition that affects the interoperability of applications. Furthermore Standard BDPs can be broken down into the following categories.

Core ({{site.pages|where:"type","Standards Track"|where:"category","Core"|size}})

Improvements requiring a core implementation.

Interface ({{site.pages|where:"type","Standards Track"|where:"category","Interface"|size}})

Includes improvements around client API/RPC specifications and standards.
